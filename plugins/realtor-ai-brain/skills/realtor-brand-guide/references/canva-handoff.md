# Canva Designer Handoff Brief

This reference defines the exact format for the Canva/Fiverr designer handoff brief. Generate this as a separate DOCX file after completing the HTML brand guide. The brief must be complete enough that a designer can execute without asking a single follow-up question.

---

## When to Generate

Always generate after delivering the HTML brand guide. The agent gets two files:
1. `[name]-brand-guide.html` — the full brand style sheet (for their own reference and team)
2. `[name]-canva-brief.docx` — the structured designer handoff brief (to send directly to any Canva or Fiverr designer)

---

## Brief Structure

The DOCX brief uses this exact structure. Every section must be complete — no "TBD" or "ask the agent." All decisions are made in the brand guide and carried forward here.

```
TYPE SIZES BY ELEMENT:
```

### SECTION 7: What NOT to Do

```
BRAND RULES FOR THE DESIGNER

COLORS:
✕ Do not use any Canva default or suggested colors
✕ Do not add gradients unless specifically specified in this brief
✕ Do not use transparent versions of brand colors unless specified (opacity values are intentional)

FONTS:
✕ Do not substitute fonts — if a font isn't in Canva, flag it before proceeding
✕ Do not use more than 2 font families in any design
✕ Do not use decorative or script fonts not specified here

LOGO:
✕ Do not stretch, rotate, recolor, or add effects to the logo mark
✕ Do not place the logo on a busy background without a solid backing shape
✕ Do not make the logo smaller than specified minimums

LAYOUT:
✕ Do not center-align everything — follow the hierarchy in each template spec
✕ Do not crowd text to edges — respect the safe zone margins
✕ Do not add drop shadows, glows, or 3D effects to any element

GENERAL:
✕ Do not add elements not in this brief without client approval
✕ Do not use stock photos or illustrations not approved by the client
✕ Do not use Canva's AI image generation for any brand elements
```

```
  → Business cards (if print): PDF, 300dpi, CMYK if possible, with bleed marks

REVISION PROCESS:
  → First delivery: share Canva link for review
  → Revisions: use Canva comments to mark changes needed
  → Final: export in formats listed above

QUESTIONS:
  All brand decisions are in this brief. If something is genuinely unclear,
  contact: [agent email or VA email]
  Do not proceed with assumptions — flag issues before starting.
```

## How Claude Fills This Brief

1. **Populates every bracketed field** using the exact values from the brand guide just generated — hex codes, font names, element descriptions, agent name, city, niche
2. **Converts hex to RGB** for the color table (formula: R = first two hex digits in decimal, G = middle two, B = last two)
3. **Writes precise Canva build instructions** for each graphic element based on what was designed in the HTML guide
4. **Fills the template specs** for whichever deliverables the agent requested during Step 1 intake — the template list was collected before building, never mid-production
5. **Saves as DOCX** using the docx skill — formatted cleanly with headers, tables, and the brand's primary color used for section headers
6. **Names the file** `[agent-first-last]-canva-brief.docx`
